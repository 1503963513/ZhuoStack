# 安全策略

## 支持范围

安全修复优先覆盖：

- `main` 分支上的当前代码；
- 最新的稳定 GitHub Release；
- 最近一个仍在运行的生产版本（请在报告中注明版本或 Git commit）。

旧版本通常不会单独回补安全修复，除非维护者明确发布安全公告。

## 报告漏洞

请使用 GitHub 的私密 Security Advisory 渠道：

<https://github.com/1503963513/ZhuoStack/security/advisories/new>

不要通过公开 Issue、Pull Request、Discussion 或聊天消息披露未修复漏洞。若私密入口暂时不可用，请先通过项目维护者 GitHub 主页联系：<https://github.com/1503963513>。

报告请尽量包含：

- 受影响版本、commit 或部署包名称；
- 漏洞类型、影响范围和可复现步骤；
- 最小化的 PoC、请求样例或日志；
- 可能的修复建议；
- 是否已经向其他人披露。

请删除密码、Token、Cookie、私钥、数据库连接串和真实用户数据。测试时只使用你拥有权限的环境。

## 响应承诺

维护者目标是在 3 个工作日内确认收到报告，在 7 个工作日内完成初步分级，并在确认后尽快修复、发布公告和提供升级说明。实际时间会根据漏洞复杂度、复现条件和协调披露安排变化。

确认漏洞后，维护者会：

1. 创建私密修复分支并限制知情范围；
2. 评估影响版本、严重性和是否需要撤回发布物；
3. 在修复、测试和迁移验证完成后发布新版本；
4. 在 GitHub Security Advisory 和 `CHANGELOG.md` 中记录影响、修复版本和缓解措施。

## 使用建议

- 生产环境使用最新稳定版本，并订阅 GitHub Release/安全公告；
- 不要把 `.env.deploy`、`apps/api/.env.production` 或云厂商密钥提交到仓库；
- 生产环境启用 HTTPS、强 JWT/Redis/数据库密码和最小权限对象存储密钥；
- 数据库迁移前备份并验证恢复流程；
- 对外暴露 Swagger、管理接口和监控接口前配置网络访问控制。
